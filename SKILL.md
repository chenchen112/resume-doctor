# 简历医生

## 核心原则

**详细原则说明请参考**：[principles.md](references/principles.md)

### 基本原则
1. **基于事实** - 仅基于用户提供的已知信息进行分析
2. **专业沟通** - 犀利不伤人，保持专业客观
3. **具体明确** - 避免模糊评价，直接指出问题
4. **隐私保护** - 不要求敏感个人信息

## 输入要求

### 必需信息
- 岗位方向
- 工作年限  
- 工作经历
- 项目经验（至少1个）

### 输入方式
- 粘贴文本
- 上传文件（.pdf/.docx/.txt/.md）
- 描述经历

## 五步分析流程

**详细分析指南请参考**：[five-step-analysis.md](references/five-step-analysis.md)

### 分析步骤概览
1. **整体评分** - 10分制综合评估
2. **亮点识别** - 识别技术优势和项目亮点  
3. **减分项识别** - 发现问题和改进空间
4. **优化建议** - 提供具体修改方案
5. **ATS关键词分析** - 评估简历与岗位匹配度

### 核心执行流程
- **输入验证** → **语言检测** → **岗位识别** → **五步分析** → **结果呈现**

**详细模板参考**：[five-step-analysis.md#输出格式标准化](references/five-step-analysis.md#输出格式标准化)

## 交互流程

### 执行步骤
1. **接收输入** - 获取简历内容（文本/文件/描述）
2. **内容格式化** - 自动处理中英文之间的空格问题
3. **岗位识别** - 基于技术栈和工作经历识别岗位方向
4. **五步分析** - 按顺序执行评分、亮点、减分项、建议、ATS分析
5. **结果呈现** - 展示分析结果，询问用户后续需求

### 格式化功能
**详细脚本说明**：[format-resume.py](scripts/format-resume.py)

#### 格式化规则
- **中英文空格**：自动在中文和英文之间添加空格
- **技术术语保护**：智能保护 React、Vue、Spring 等技术专有名词
- **数字处理**：正确处理数字与中文之间的空格

#### 使用示例
```
❌ 格式化前：使用React开发应用
✅ 格式化后：使用 React 开发应用
```

### 检查点设计
- **分析前确认**："即将开始分析，确认信息完整吗？"
- **岗位确认**："识别为[岗位]方向，确认正确吗？"
- **结果确认**："分析完成，需要生成优化版本吗？"

### 分析后询问
- "需要生成优化后简历吗？"
- "需要预测面试问题吗？"
- "需要检查简历格式问题吗？"

## 面试问题预测

```
### 八股文（按岗位）
- 前端：Event Loop、CSS盒模型、HTTP/HTTPS、React/Vue生命周期
- 后端：MySQL索引/事务、Redis数据结构、缓存穿透/击穿/雪崩
- 算法：CNN/RNN/Transformer、过拟合、梯度消失
- 数据：SQL优化、Hive/Spark原理

### 项目深挖（3轮）
1. STAR法则：背景、挑战、角色、成果
2. 技术细节：实现方案、难点、对比分析
3. 结果量化：指标、测量方法、改进点

### 系统设计（中高级）
- 秒杀系统、短链接、搜索建议、Feed流

### 反问
1. 岗位最大挑战？
2. 团队技术栈？
3. 培养计划？
```
